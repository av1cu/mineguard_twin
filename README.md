# MineGuard Twin ─ Цифровой двойник карьера с AI-диспетчером

**MineGuard Twin** — это интеллектуальная система промышленной безопасности и оптимизации логистики открытого карьера. Система объединяет симуляцию движения карьерной техники, энергетический аудит рейсов и компьютерное зрение (CV) для контроля устойчивости бортов карьера и состояния водителей в единую среду принятия решений диспетчера.

---

## 🏗️ Архитектура системы

Система разработана с четким разделением на независимые слои:

```
                    ┌─────────────────────────┐
                    │     Streamlit Web UI    │ (Порт 8501)
                    └────────────┬────────────┘
                                 │ HTTP / REST API
                    ┌────────────▼────────────┐
                    │   FastAPI API Backend   │ (Порт 8000)
                    └────────────┬────────────┘
                                 │ SQLite DB
        ┌────────────────────────┼────────────────────────┐
        │                        │                        │
┌───────▼────────┐      ┌────────▼─────────┐      ┌───────▼─────────┐
│  MineOps Twin  │      │  Slope/Road CV   │      │    Driver CV    │
│  (OpenMines)   │      │     Monitor      │      │     Monitor     │
└────────────────┘      └──────────────────┘      └─────────────────┘
```

1. **Фронтенд (Web UI Dashboard)**:
   * Реализован на **Streamlit** (модули `app/main.py` и `app/pages/*`).
   * Отображает интерактивную Plotly-карту карьера с движущимися самосвалами (Digital Twin).
   * Выводит графики сравнения Baseline и Optimized сценариев.
   * Позволяет загружать кадры бортов и обрабатывать веб-камеру водителя.

2. **Бэкенд (FastAPI Backend)**:
   * Реализован на **FastAPI** (`backend/api.py`).
   * Хранит состояние в легковесной базе данных SQLite.
   * Предоставляет REST-эндпоинты для регистрации событий, получения KPI и обновления координат.

3. **Симуляция и слои аналитики (Simulation & Energy/Safety Layers)**:
   * **OpenMines Adapter** (`simulation/openmines_adapter.py`) — запускает симуляцию карьера, рассчитывает перемещения техники по дорожной сети.
   * **Energy Layer** (`simulation/energy_layer.py`) — рассчитывает расход топлива (с грузом, пустой, холостой ход) и энергозатраты в кВт/ч.
   * **Safety Layer** (`simulation/safety_layer.py`) — отслеживает опасные сближения самосвалов.
   * **Smart Dispatcher** (`simulation/smart_dispatcher.py`) — перестраивает маршруты при обнаружении аномалий.

4. **CV-Мониторинг**:
   * **Slope/Road CV** (`cv_slope/detect_slope.py`) — детекция трещин и оползней с использованием классического Canny-анализа OpenCV и симуляции разметки.
   * **Driver Fatigue CV** (`cv_driver/fatigue_detector.py`) — MediaPipe Face Mesh для расчета EAR (закрытие глаз) и MAR (зевота) по камере.

## 🚀 Инструкция по запуску

### Вариант 1. Запуск через Docker Compose (Рекомендуемый)

Убедитесь, что у вас установлен Docker и Docker Compose, после чего выполните:

```bash
docker-compose up --build
```

* **Web UI Dashboard** будет доступен по адресу: [http://localhost:8501](http://localhost:8501)
* **FastAPI Swagger API Docs** будет доступен по адресу: [http://localhost:8000/docs](http://localhost:8000/docs)

---

### Вариант 2. Ручной локальный запуск

#### 1. Установка зависимостей
Требуется Python 3.9+. Создайте виртуальное окружение и установите библиотеки:

```bash
python -m venv venv
# Активация окружения (Windows)
venv\Scripts\activate
# Активация окружения (Linux/macOS)
source venv/bin/activate

pip install -r requirements.txt
```

#### 2. Инициализация базы данных
Инициализируйте SQLite базу данных и таблицы:

```bash
python backend/database.py
```

#### 3. Запуск Бэкенда
Запустите API-сервер FastAPI:

```bash
uvicorn backend.api:app --host 0.0.0.0 --port 8000
```

#### 4. Запуск Фронтенда
В новом терминале (активировав виртуальное окружение) запустите Streamlit:

```bash
streamlit run app/main.py
```

Откройте в браузере [http://localhost:8501](http://localhost:8501).

## 🎬 Демонстрационный сценарий для защиты (Вау-эффект)

Для демонстрации полной интеграции модулей жюри:

1. **Шаг 1: Запуск базовой симуляции**
   * Перейдите во вкладку **02 Simulation**.
   * Нажмите кнопку **Запустить Baseline симуляцию**. Дождитесь завершения.
   * Перейдите во вкладку **01 Dashboard**, нажмите **Начать мониторинг** и посмотрите, как самосвалы ездят по стандартным маршрутам.

2. **Шаг 2: Регистрация аномалии борта карьера**
   * Перейдите во вкладку **03 Slope CV**.
   * Выберите изображение `Опасный участок (осыпь, трещина возле дороги)`.
   * Модель обнаружит трещину и оползень, присвоив высокий Risk Score.
   * Нажмите красную кнопку **Отправить событие в цифровой двойник**.

3. **Шаг 3: Автоматическое перестроение маршрутов**
   * Перейдите во вкладку **02 Simulation** и запустите **Optimized симуляцию**.
   * На карте **01 Dashboard** вы увидите, что участок дороги **ROUTE-02** заблокирован (горит красным), а умный диспетчер перенаправил самосвалы (например, `TRUCK-02` теперь едет в объезд по резервной дороге).
   * Графики во вкладке **02 Simulation** наглядно покажут снижение расхода топлива на тонну и отсутствие опасных сближений по сравнению с Baseline.

4. **Шаг 4: Детекция микросна водителя**
   * Перейдите во вкладку **04 Driver CV**.
   * Выберите **Симулятор состояния водителя** (или включите Веб-камеру).
   * Отметьте флажок **Водитель закрыл глаза (Микросон)**.
   * Нажмите **Отправить CV-событие на бэкенд**.
   * Вернитесь на **01 Dashboard** или **05 Events** — вы увидите, что `TRUCK-04` мгновенно получил уровень риска **CRITICAL**, его скорость упала до 0 км/ч (остановлен ради безопасности), а диспетчеру выдана рекомендация: *"Stop truck immediately"*.
